# EXPECTED OUTPUTS - LYDO CONNECT

## Overview

This section presents the expected outputs of the study based on the actual scope of LYDO Connect: Youth Information, Engagement, and Transparency Management System and the current direction of the concept paper. The outputs are aligned with the project's objective of developing a centralized web-based Progressive Web Application (PWA) that supports youth information dissemination, participation monitoring, transparency reporting, and administrative management for selected Local Youth Development Offices within Metro Manila.

The expected outputs are grouped into primary deliverables, which refer to the core system components to be produced, and secondary deliverables, which refer to the supporting documents and evaluation outputs needed for proper implementation, use, and assessment of the system.

## Primary Deliverables

### 1. LYDO Connect System

The primary output of the study is a functional LYDO Connect system developed as a centralized web-based Progressive Web Application (PWA). The system is expected to serve as the main digital platform for managing youth information, participation records, transparency-related content, and citizen-facing services in a more organized and accessible environment.

### 2. Public Youth and Transparency Portal

The study is expected to produce a public-facing portal where users can access youth programs, events, organizations, participation-related information, transparency records, financial disclosure content, barangay-based information, service advisories, and citizen desk services. This portal is intended to provide youth participants and the public with a centralized access point for LYDO-related information and services.

### 3. Administrative Portal

The study is also expected to produce an administrative portal for authorized LYDO staff and system administrators. This portal will support the management of programs, events, organizations, registrations, transparency documents, compliance records, financial data, citizen concerns, users, roles, and audit logs. It is intended to improve record management, monitoring, and accountability on the administrative side of the system.

### 4. Shared Database System

Another primary output is a centralized shared database that stores and organizes the system's major records, including user accounts, youth programs, events, registrations, organizations, transparency documents, financial and compliance records, citizen desk entries, advisories, and audit-related data. This database is expected to support consistent data management across both the public and administrative components of the platform.

## Secondary Deliverables

### 1. Technical Documentation

The project is expected to produce technical documentation covering the system architecture at a general level, database structure, module descriptions, system configuration requirements, and key implementation components. This documentation will support maintenance, turnover, and future enhancement of the system.

### 2. Project Documentation

The study is expected to generate complete project documentation in the form of the capstone manuscript and related development records. This includes the problem context, objectives, scope and limitations, legal and literature basis, methodology, system description, and other academic requirements relevant to the project.

### 3. User Manual and Administrator Guide

The project is expected to provide user-oriented documentation that explains how the system is used by both public users and administrators. This includes instructions for account access, profile use, program and event participation, transparency access, citizen desk usage, and administrative record management.

### 4. System Evaluation Results

The study is expected to produce system evaluation results based on the selected assessment framework of the project, particularly the ISO/IEC 25010 software quality model identified in the paper. These results are expected to show the quality and acceptability of the system in terms of relevant criteria such as functionality, usability, reliability, performance efficiency, security, and maintainability.

### 5. Deployment and Configuration Guide

The project is expected to include a deployment and configuration guide that outlines the basic setup requirements for running the system in its intended environment. This may include hosting requirements, account and access setup, environment configuration, and operational reminders for pilot use or turnover.

### 6. Source Code and Supporting System Assets

The project is expected to produce the source code, interface assets, database-connected components, and related configuration files needed to operate the system. These materials form part of the technical output of the study and support future maintenance, review, and enhancement of the project.

## Concluding Expected Output Statement

In summary, the expected outputs of the study include a functional LYDO Connect platform composed of a public portal, an administrative portal, and a shared database system, together with the necessary technical, user, academic, evaluation, and deployment documents. These outputs are intended to demonstrate not only the completion of a working information system, but also the project's readiness for academic evaluation, pilot implementation, and future improvement in the context of Metro Manila LYDO operations.
