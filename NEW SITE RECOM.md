# NEW SITE RECOM

## Recommended New Direction

The site should be redesigned as an **organization-centric LYDO transaction and transparency platform**, not only an information website.

The most important shift is this:

- from viewing only
- to document submission, validation, monitoring, and public disclosure

## Core Concept

The new site should support the full flow of an organization's interaction with LYDO:

1. organization profile setup
2. document submission
3. validation and review
4. budget request or allocation tracking
5. budget utilization monitoring
6. liquidation and reporting
7. public transparency posting
8. compliance status updates
9. remarks and consequences for missing or invalid submissions

## Suggested Site Structure

### Public Side

For youth, visitors, and the general public.

#### 1. Home

Purpose:

- introduce the platform
- show the covered organizations
- show the LYDO role
- show latest transparency highlights

Usage:

- first landing page

#### 2. About

Purpose:

- explain the mission of the site
- define what the platform covers
- define what transparency means in the system

Usage:

- helps users understand the scope before browsing

#### 3. Organizations

Purpose:

- display organizations handled by LYDO
- show type, status, recognition, barangay, and coverage

Usage:

- public directory
- reference page for youth and partners
- org-centric browsing

#### 4. Programs and Projects

Purpose:

- show official projects, activities, and PPAs
- link projects to organizations and budgets

Usage:

- transparency and participation reference
- public view of what each organization is doing

#### 5. Transparency Board

Purpose:

- publish compliance updates
- show disclosure documents
- show budget transparency records

Usage:

- public accountability page
- supports the transparency requirement of the panel

#### 6. Financial Disclosure

Purpose:

- show budget-related summaries
- show utilization and liquidation records where allowed

Usage:

- public visibility of official financial records

#### 7. Youth Desk / Inquiry Portal

Purpose:

- receive complaints
- receive inquiries
- receive follow-up concerns from youth and orgs

Usage:

- communication channel between youth and LYDO

### Organization Portal

This is the part that should make the system truly transaction-based.

#### 8. Organization Dashboard

Purpose:

- let organizations see their status
- show pending requirements
- show submitted documents
- show budget and compliance progress

Usage:

- organization self-service center

#### 9. Document Submission

Purpose:

- upload requirements digitally
- submit forms, reports, and supporting files

Usage:

- replaces manual file submission
- main transaction feature for organizations

#### 10. Budget Request Tracker

Purpose:

- show budget request status
- show approval status

Usage:

- makes budget-related workflow visible

#### 11. Utilization and Liquidation Tracker

Purpose:

- record how funds were spent
- attach receipts, summaries, and liquidation documents

Usage:

- shows whether the budget was used properly
- addresses the panel's concern about misuse

#### 12. Compliance Status Page

Purpose:

- show whether the organization is fully compliant, partially compliant, pending, or missing requirements

Usage:

- quick status reference for both orgs and LYDO

### Admin Side

Usage:

Purpose:

Purpose:

Usage:

Usage:

#### 17. Compliance and Sanction Tracking

Purpose:

- record late submissions

Usage:

- shows consequences of non-compliance
- makes monitoring realistic

#### 18. Reports and Export Module

Purpose:

- generate PDF, CSV, or summary reports

Usage:

#### 19. Audit Log

Purpose:

- record who changed what and when
- support accountability

Usage:

- useful for transparency and admin tracking

## Must-Have Features

### 1. Digital document submission

Why:

- this is the core transaction the panel wants

Use:

- upload requirements, reports, receipts, and attachments

### 2. Status tracking

Why:

- organizations need to know whether their submissions are complete

Use:

- pending, approved, rejected, incomplete, returned for revision

### 3. Budget transparency tracker

Why:

- the panel kept asking how the budget will be monitored

Use:

- released amount
- utilized amount
- remaining amount
- remarks

### 4. Compliance dashboard

Why:

- makes fully compliant / partially compliant meaningful

Use:

- per organization compliance view

### 5. Public disclosure board

Why:

- satisfies transparency

Use:

- publish official records and summaries

### 6. Remarks and deficiency tracking

Why:

- the panel asked what happens when documents are missing

Use:

- record comments, missing items, and follow-up actions

### 7. Organization classification

Why:

- the panel asked what kind of organization is being handled

Use:

- SK
- school-based
- barangay-based
- private youth organization
- youth-serving partner

### 8. Role-based access

Why:

- different users should not see the same functions

Use:

- public view
- organization view
- admin view

### 9. Notification system

Why:

- to remind orgs about missing or upcoming requirements

Use:

- submission reminders
- approval updates
- compliance notices

### 10. Search and filtering

Why:

- users should quickly find organizations and records

Use:

- filter by status, type, barangay, and compliance

## Recommended Workflow Flow

### For organizations

1. Register or log in
2. Open organization dashboard
3. Check required documents
4. Upload files
5. Track review status
6. Upload revisions if needed
7. View budget allocation or request status
8. Submit liquidation or accomplishment report
9. Wait for compliance update

### For admin

1. Receive submission
2. Check completeness
3. Validate files
4. Add remarks
5. Approve or return for revision
6. Record budget status
7. Update compliance status
8. Publish public disclosure if allowed

## What To Remove Or Rework From The Current Direction

The current concept should be reduced or reworked if it only does the following:

- shows organization profiles without workflow
- shows announcements only
- acts like a directory only
- uses individual youth as the main transaction user
- lacks budget and liquidation visibility

Those parts should either be removed, separated into a smaller feature, or made secondary.

## Recommended New Site Identity

The best identity for the revised site is:

**A web-based organization management, compliance, and transparency system for LYDO**

This is stronger than calling it only a:

- youth portal
- information site
- directory
- static PWA

## Final Recommendation

If you are reworking the site, prioritize these in order:

1. organization-centric workflow
2. document submission
3. compliance validation
4. budget tracking
5. liquidation and utilization tracking
6. public transparency board
7. admin remarks and consequences
8. role-based access and reporting

That will make the project much closer to what the panel wants and much easier to defend.
