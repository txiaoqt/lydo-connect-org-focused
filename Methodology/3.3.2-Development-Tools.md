# 3.3.2 Development Tools

The system development tools used in the current LYDO Connect implementation include **TypeScript**, **JavaScript**, and **SQL**. The web application was developed using **React** and **Vite**, with **TypeScript** as the main application language and **SWC** through the Vite React plugin for fast local builds. Interface development used **Tailwind CSS**, **shadcn/ui**, **Radix UI**, and **Lucide React** to produce a consistent component-based user interface.

For application behavior, the project uses **React Router** for route management, **TanStack Query** and React state/context patterns for data loading and client-side state handling, and **React Hook Form** with **Zod** and resolver helpers for form processing and validation. Some current workflows also rely on libraries such as **PDF.js** and **Tesseract.js** for document preview and OCR-related handling within the portal.

The backend and data services are handled through **Supabase**, particularly **Supabase Auth**, **PostgREST-based APIs**, **Supabase Storage**, and **SQL RPC functions**, with **Supabase PostgreSQL** serving as the primary database for structured records. The project also includes SQL-based schema management for tables, constraints, relationships, and role-aware access behavior that support the implemented workflow.

For quality assurance and project operations, the development environment is **Visual Studio Code**, while **ESLint** and **Vitest** are used for linting and testing. **Git** and **GitHub** support version control and collaboration, and deployment is handled through **Vercel** for the frontend together with **Supabase cloud services** for backend infrastructure and managed storage.
