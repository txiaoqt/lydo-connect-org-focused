# 3.3.2 Development Tools

The following tools and technologies are used in the development of LYDO Connect. These tools are appropriate because they support responsive web development, cloud-backed data management, rapid interface prototyping, and modular deployment.

## Frontend Development Tools

| Tool | Purpose in the Study |
|---|---|
| React 18 | Builds the interactive public portal and admin portal interfaces |
| TypeScript | Improves code reliability through typed frontend logic |
| Vite | Provides fast development server startup and optimized build generation |
| React Router DOM | Handles route-based navigation for public and admin pages |
| TanStack Query | Supports structured client-side data fetching and caching |
| Tailwind CSS | Provides utility-based styling for responsive page layout |
| ShadCN UI and Radix UI components | Supplies reusable interface primitives for forms, dialogs, tables, and navigation |
| Recharts | Supports chart-based financial and dashboard visualizations |
| React Leaflet and Leaflet | Supports map-based views and barangay location display |

## Backend and Database Tools

| Tool | Purpose in the Study |
|---|---|
| Supabase | Provides backend-as-a-service features for authentication, PostgreSQL database access, storage, and RPC execution |
| PostgreSQL | Stores structured data for users, youth records, transparency records, ticketing, and audit logs |
| Supabase Storage | Stores uploaded transparency files and related public document assets |
| SQL migrations | Defines tables, constraints, triggers, functions, and access policies in versioned form |

## Integration and Automation Tools

| Tool | Purpose in the Study |
|---|---|
| Python | Supports worker-side automation for registration sync processing |
| Requests library | Sends HTTP requests to Google Forms and reads external endpoints where needed |
| Google Forms and Google Sheets | Serve as optional external attendance collection and reporting targets |
| OpenStreetMap and Nominatim | Support map display, location lookup, and route-related features |
| Facebook source links | Provide official source-post references for program and event records |

## Development and Quality Tools

| Tool | Purpose in the Study |
|---|---|
| Visual Studio Code | Primary code editor for development and debugging |
| ESLint | Enforces code quality and consistency rules |
| Vitest | Supports unit and component-level testing for the frontend |
| Git | Supports version control and iterative code management |

## Deployment Tools

| Tool | Purpose in the Study |
|---|---|
| Vercel | Hosts the web application and optional serverless sync trigger endpoint |
| Progressive Web App manifests | Supports installable web experience, particularly for the admin surface |

## Summary

These development tools support the study's need for a modern, responsive, maintainable, and deployable web-based governance platform. They also match the actual implementation present in the repository.
