# 3.2.1 Activity Diagram

Activity diagrams show how the major LYDO Connect workflows proceed from one action to the next. The diagrams below focus on the implemented portal-local workflows for registration, citizen services, transparency publication, compliance records, and audit-supported administration.

## Figure 6.1. Activity Diagram of Program or Event Registration

```mermaid
flowchart TD
    A([Start]) --> B[Youth user opens a program or event record]
    B --> C{Is the user authenticated?}
    C -- No --> D[Prompt user to sign in or create an account]
    D --> B
    C -- Yes --> E[Display registration form]
    E --> F[User enters name, email, contact number, municipality, and barangay]
    F --> G[System validates required fields and contact format]
    G -- Invalid --> H[Display validation message]
    H --> E
    G -- Valid --> I{Record type}
    I -- Event --> J[Check event status and capacity]
    I -- Program --> K[Check program publication status]
    J -- Closed or full --> L[Display registration unavailable message]
    K -- Not published --> L
    J -- Available --> M[Create or update event registration]
    K -- Published --> N[Create or update program registration]
    N --> O[Create or reactivate program membership]
    M --> P[Display successful registration confirmation]
    O --> P
    P --> Q[Admin or staff reviews local registration records]
    Q --> R([End])
    L --> R
```

## Figure 6.2. Activity Diagram of Citizen Desk Submission and Tracking

```mermaid
flowchart TD
    A([Start]) --> B[Citizen opens the Citizen Desk page]
    B --> C[Select ticket type and enter concern details]
    C --> D[System validates ticket subject, message, email, and contact data]
    D -- Invalid --> E[Display input errors]
    E --> C
    D -- Valid --> F[Create citizen ticket record]
    F --> G[Generate public reference number]
    G --> H[Display reference number and received status]
    H --> I[Citizen searches by reference number and email]
    I --> J{Matching ticket found?}
    J -- No --> K[Show no matching ticket message]
    J -- Yes --> L[Display ticket status and timestamps]
    L --> M[Admin or staff reviews ticket queue]
    M --> N[Update status, priority, or assigned user]
    N --> O[Audit log records administrative change]
    O --> P[Updated ticket details appear in tracking results]
    P --> Q([End])
    K --> Q
```

## Figure 6.3. Activity Diagram of Transparency and Governance Publication

```mermaid
flowchart TD
    A([Start]) --> B[Admin or staff opens governance module]
    B --> C{Selected module}
    C -- Documents --> D[Enter document metadata and upload public file]
    C -- Financials --> E[Encode barangay financial values]
    C -- Compliance --> F[Encode quarterly or monthly compliance status]
    C -- Advisories --> G[Create public service advisory]
    D --> H[Validate year, quarter, barangay scope, office, and file metadata]
    E --> I[Validate fiscal year, month, and amount ranges]
    F --> J[Validate reporting period and status values]
    G --> K[Validate advisory title, message, office, and active dates]
    H -- Invalid --> L[Display validation error]
    I -- Invalid --> L
    J -- Invalid --> L
    K -- Invalid --> L
    L --> B
    H -- Valid --> M[Save disclosure document record]
    I -- Valid --> N[Save financial or youth metric record]
    J -- Valid --> O[Save compliance record]
    K -- Valid --> P[Save service advisory record]
    M --> Q[Audit log records administrative action]
    N --> Q
    O --> Q
    P --> Q
    Q --> R[Public portal displays updated records]
    R --> S([End])
```

## Interpretation

- The registration workflow stores participation directly in `event_registrations` or `program_registrations`, with program registrations also maintaining `user_program_memberships`.
- The citizen desk workflow supports public tracking through `citizen_tickets` and administrative handling through assigned users, status updates, and audit logs.
- The transparency workflow shows how administrative records become public-facing governance information through disclosure, financial, compliance, and advisory modules.
