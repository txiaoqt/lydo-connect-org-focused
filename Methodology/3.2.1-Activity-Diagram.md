# 3.2.1 Activity Diagram

The activity diagrams show the major workflows that currently exist in LYDO Connect.

## Figure 6.1. Authentication and Policy Agreement

```mermaid
flowchart TD
    A([Start]) --> B["Open LYDO Connect"]
    B --> C["Sign up or sign in"]
    C --> D["Validate credentials"]
    D -- Invalid --> E["Show error"]
    D -- Valid --> F{"Non-admin user?"}
    F -- Yes --> G["Load active policy"]
    G --> H{"Policy accepted?"}
    H -- No --> I["Show policy agreement"]
    I --> H
    H -- Yes --> J["Open user portal"]
    F -- No --> K["Open admin portal"]
    J --> L([End])
    K --> L
```

## Figure 6.2. User Workflow

```mermaid
flowchart TD
    A([Start]) --> B["Open user portal"]
    B --> C["Manage organization profile"]
    C --> D["Submit required documents"]
    D --> E["Create or edit budget request"]
    E --> F["Upload supporting budget file"]
    F --> G{"Budget approved?"}
    G -- No --> H["Wait for approval"]
    G -- Yes --> I["Open liquidation record"]
    I --> J["Upload post-activity files"]
    J --> K["View news releases and transparency posts"]
    K --> L["Check notifications"]
    L --> M([End])
```

## Figure 6.3. Admin Workflow

```mermaid
flowchart TD
    A([Start]) --> B["Open admin portal"]
    B --> C["Review organization profiles and documents"]
    C --> D["Approve, revise, or reject records"]
    D --> E["Monitor budget requests"]
    E --> F["Update budget status"]
    F --> G["Monitor liquidation records"]
    G --> H["Update liquidation status"]
    H --> I["Review barangay budget allocation summaries"]
    I --> J["Create or edit news releases"]
    J --> K["Manage transparency posts and required document types"]
    K --> L["Review notifications and activity logs"]
    L --> M([End])
```

## Interpretation

- The current workflow is centered on authenticated user tasks and admin review tasks.
- Budget approval unlocks liquidation file uploads.
- Admin budget monitoring includes released amounts grouped by barangay and district.
- News releases are previewable before opening the Facebook source post.
- Transparency posts, required document types, notifications, and logs are maintained from the admin side.
