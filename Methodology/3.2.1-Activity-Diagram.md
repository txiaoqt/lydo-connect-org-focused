# 3.2.1 Activity Diagram

The activity diagrams show the major workflows that currently exist in LYDO Connect.

## Figure 6.1. Authentication and Policy Agreement

```mermaid
flowchart TD
    A([Start]) --> B["Open LYDO Connect"]
    B --> C["Open sign-in page"]
    C --> D["Enter credentials"]
    D --> E["Authenticate account"]
    E -- Invalid --> F["Display login error"]
    F --> D
    E -- Valid --> G{"Admin account?"}
    G -- Yes --> H["Open admin portal"]
    G -- No --> I["Load active policy"]
    I --> J{"Policy already accepted?"}
    J -- No --> K["Show policy agreement"]
    K --> L{"Accept policy?"}
    L -- No --> M["Keep user out of protected pages"]
    M --> K
    L -- Yes --> N["Record policy acceptance"]
    J -- Yes --> O["Open user portal"]
    N --> O
    H --> P([End])
    O --> P
```

## Figure 6.2. User Workflow

```mermaid
flowchart TD
    A([Start]) --> B["Open user portal"]
    B --> C["Review dashboard status"]
    C --> D["Create or update organization profile"]
    D --> E["Open compliance document submission"]
    E --> F["Upload, preview, and submit required files"]
    F --> G["Create, edit, or submit budget request"]
    G --> H{"Budget released?"}
    H -- No --> I["Monitor request status and notifications"]
    I --> J{"Open YPOP workflow?"}
    H -- Yes --> K["Open liquidation reporting"]
    K --> L["Upload liquidation files and submit report"]
    L --> I
    J -- Yes --> M["Create or update YPOP entry"]
    M --> N["Upload YPOP proof files and submit entry"]
    N --> O["Check YPOP validation status"]
    O --> P["Browse news releases and portal updates"]
    J -- No --> P
    P --> Q([End])
```

## Figure 6.3. Admin Workflow

```mermaid
flowchart TD
    A([Start]) --> B["Open admin portal"]
    B --> C["Review overview cards and pending counts"]
    C --> D["Open registration and document review"]
    D --> E["Approve, revise, reject, or update profile and compliance records"]
    E --> F["Open budget request review"]
    F --> G["Approve, revise, reject, or release budget records"]
    G --> H["Open liquidation monitoring"]
    H --> I["Approve, revise, or mark liquidation status"]
    I --> J["Open budget tracking and barangay allocation monitoring"]
    J --> K["Review monitoring summary and filtered allocation results"]
    K --> L["Open YPOP validation and YORP registry"]
    L --> M["Manage YPOP cycles, validation decisions, and registry records"]
    M --> N["Open news releases and templates"]
    N --> O["Create, update, publish, hide, or remove content records"]
    O --> P["Review notifications and activity logs"]
    P --> Q([End])
```

## Interpretation

- The current workflow is centered on authenticated organization-side operations and administrative review operations.
- Policy agreement remains a gating step for non-admin users after successful authentication.
- Organization users move from profile completion to compliance submissions, budget requests, liquidation reporting, YPOP processing, notifications, and news updates.
- Admin users move from overview monitoring to registration review, finance-related review, YPOP and YORP management, content maintenance, and audit-oriented monitoring.
- Budget tracking still includes released amounts and allocation summaries grouped by district and barangay.
