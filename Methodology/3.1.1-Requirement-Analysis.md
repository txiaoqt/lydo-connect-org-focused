# 3.1.1 Requirement Analysis

Requirement analysis identifies what LYDO Connect must do, who will use it, and what conditions the system must satisfy. In the current application, the scope centers on sign-up, sign-in, sign-out, policy agreement gating, organization profile management, document submission, budget request CRUD, liquidation reporting after approval, barangay-level budget allocation monitoring, news releases, public transparency posts, notifications, required document type management, and admin review.

## Stakeholders and Their Needs

| Stakeholder | Primary Need | System Response |
|---|---|---|
| Organization users | Manage profile data, submit documents, request budget, upload liquidation files, and track notifications | User portal with authentication, policy agreement gating, profile, document submission, budget request, liquidation, news release preview, transparency, and notification screens |
| Public visitors | View published news and public transparency content | Public-facing news release and transparency pages |
| LYDO administrators | Review submissions, approve or reject records, manage content, and maintain document types | Admin portal with overview, document review, budget monitoring, barangay allocation summaries, liquidation monitoring, content CRUD, required document types, notifications, and activity logs |
| System evaluators | Check usability, reliability, and traceability | Logged actions, consistent status records, and clear workflow screens |

## Functional Requirements

| ID | Requirement | Description |
|---|---|---|
| FR-01 | User authentication | The system shall allow users to sign up, sign in, and sign out. |
| FR-02 | Policy agreement | The system shall require non-admin users to accept the active Terms of Service and Privacy Policy before protected workflows continue. |
| FR-03 | Organization profile management | The system shall allow organization users to create and update their organization profile. |
| FR-04 | Document submission | The system shall allow organization users to upload required documents and preview submitted files. |
| FR-05 | Budget request CRUD | The system shall allow organization users to create, edit, update, delete, save draft, and submit budget requests. |
| FR-06 | Budget request attachments | The system shall allow budget requests to include a detailed supporting file upload. |
| FR-07 | Liquidation access control | The system shall make liquidation available only after a related budget request is approved. |
| FR-08 | Liquidation file upload | The system shall allow post-activity documents such as photos and attendance sheets to be uploaded to liquidation records. |
| FR-09 | News release preview | The system shall allow users and admins to preview news releases in a dedicated page before opening the source Facebook post. |
| FR-10 | Public transparency posting | The system shall allow administrators to manage transparency posts visible to the public. |
| FR-11 | Required document type management | The system shall allow administrators to manage required document type records and template files. |
| FR-12 | Notifications | The system shall record and display workflow-related notifications to users. |
| FR-13 | Admin review | The system shall allow admins to review organization registrations, documents, budget requests, liquidations, transparency posts, and news releases. |
| FR-14 | Audit logging | The system shall preserve important admin actions in activity logs. |
| FR-15 | Barangay budget monitoring | The system shall group released budget amounts by barangay and district for administrative monitoring. |

## Non-Functional Requirements

| ID | Requirement | Description |
|---|---|---|
| NFR-01 | Usability | The interface shall be understandable to youth organization users and LYDO staff. |
| NFR-02 | Responsiveness | The system shall work on desktop and mobile browsers through responsive design. |
| NFR-03 | Reliability | The system shall preserve records consistently during create, update, delete, and review operations. |
| NFR-04 | Security | The system shall protect accounts and records through authenticated and role-based access. |
| NFR-05 | Data integrity | Validation rules shall prevent missing or inconsistent workflow data. |
| NFR-06 | Maintainability | The solution shall remain modular so that user and admin workflows can be updated independently. |
| NFR-07 | Traceability | Important administrative changes shall be traceable through logs and status records. |
| NFR-08 | Filtering and summary performance | The budget monitoring view shall support barangay and district filters without slowing the admin workflow. |

## Requirement Summary

The requirement analysis shows that LYDO Connect functions as a focused compliance and content management platform. The current workflow is centered on authenticated access, policy acceptance, organization records, document handling, budget and liquidation processing, barangay-level budget monitoring, news and transparency publishing, notifications, required document type management, and admin review. Modules not present in the current application workflow are excluded from the scope.
