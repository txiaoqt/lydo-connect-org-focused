# 3.5 System Requirements

This section presents the hardware, software, network, and access requirements needed to develop, deploy, and use LYDO Connect. Because the system is web-based and cloud-backed, the requirements should be divided into client-side, development-side, and service-side needs. Within this scope, the organization module provides verified youth organization profiles, source-backed references, and initiative context; it no longer provides user enrollment or join actions. The current site also requires support for policy agreement records, registration monitoring, and optional Google Forms/Sheets registration sync.

## 3.5.1 Hardware Requirements

| Environment | Minimum Requirement | Recommended Requirement |
|---|---|---|
| Developer workstation | Dual-core processor, 8 GB RAM, 10 GB free storage | Quad-core processor, 16 GB RAM, SSD storage |
| Admin desktop or laptop | Dual-core processor, 4 GB RAM, 1366x768 display | Quad-core processor, 8 GB RAM, Full HD display |
| Public user desktop or laptop | Dual-core processor, 4 GB RAM | Quad-core processor, 8 GB RAM |
| Mobile user device | Android or iOS smartphone with modern browser and 2 GB RAM | 4 GB RAM or better with stable mobile data or Wi-Fi |

## 3.5.2 Software Requirements

| Category | Requirement |
|---|---|
| Operating system | Windows 10 or later, macOS, or Linux for development; any modern desktop or mobile OS for end users |
| Browser | Latest or recent versions of Chrome, Edge, Firefox, Safari, or equivalent Chromium-based browser |
| Runtime for frontend development | Node.js version 18 or later |
| Runtime for registration worker | Python 3 with the worker dependencies listed in `requirements.txt` or `worker/requirements.txt` |
| Package manager | npm |
| Programming languages | TypeScript for frontend, SQL for database logic, and Python for registration sync automation |
| Frameworks and libraries | React, Vite, Tailwind CSS, Supabase client libraries, React Router, TanStack Query |
| Database platform | PostgreSQL through Supabase |
| Development tools | Visual Studio Code, Git, ESLint, Vitest |

## 3.5.3 Network and Deployment Requirements

| Requirement | Description |
|---|---|
| Internet connectivity | Required for authentication, database access, document retrieval, and cloud-hosted operation |
| HTTPS-enabled deployment | Recommended for secure public and admin access |
| Cloud backend | Requires a configured Supabase project for authentication, database, storage, and policies |
| Web hosting | Requires a deployment environment such as Vercel for frontend hosting |
| Registration sync endpoint | Requires a configured serverless route and `SYNC_TRIGGER_TOKEN` when admins use "Run Sync Now" |
| External attendance sources | Requires valid Google Forms and published Google Sheets links when external registration sync or sheet preview is enabled |
| Optional notification service | Email, push, or messaging gateway support may be required if future notification and alert features are activated |
| Map and location services | Internet access is required for map tiles and geocoding helpers |

## 3.5.4 User and Access Requirements

| User Type | Access Requirement |
|---|---|
| Public viewer | No login required for public information modules |
| Youth user | Authenticated account and active policy acceptance required for registration, profile, and participation history features; organization details remain publicly viewable through "View Organization Info" |
| Admin or staff user | Authorized role assignment required for admin modules and protected data operations |
| System maintainer | Environment variables, database migration access, deployment credentials, Supabase service role access for worker execution, and sync trigger token management required |

## 3.5.5 Data and Source Requirements

| Requirement | Description |
|---|---|
| Verified source references | Organization records shall include an official source link or citation field for credibility and auditability. |
| Organization coverage | Organization datasets shall prioritize youth organizations, youth offices, youth councils, and related youth initiatives. |
| Nullable unverifiable fields | Fields with no verifiable public data shall be stored as null or left blank instead of inferred or fabricated values. |
| Structured organization details | The database shall support name, category, overview, mission or objectives, programs or initiatives, coverage area, beneficiaries, contact details, and related activity year. |
| Policy content | The database shall include an active policy version containing Terms of Service and Privacy Policy content before authenticated user workflows are evaluated. |
| Registration sync metadata | Program, event, and registration records shall include form URLs, sheet URLs, source values, and sync lifecycle fields when external attendance integration is used. |

## Summary

These system requirements are more suitable for LYDO Connect than the template's marketplace-specific setup because they reflect a responsive cloud-based governance platform with authentication, relational storage, public transparency access, and administrative workflows.
