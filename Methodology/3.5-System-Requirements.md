# 3.5 System Requirements

This section presents the hardware, software, network, and access requirements needed to develop, deploy, and use LYDO Connect. Because the system is web-based and cloud-backed, the requirements should be divided into client-side, development-side, and service-side needs.

## 3.5.1 Hardware Requirements

| Environment | Minimum Requirement | Recommended Requirement |
|---|---|---|
| Developer workstation | Dual-core processor, 8 GB RAM, 10 GB free storage | Quad-core processor, 16 GB RAM, SSD storage |
| Admin desktop or laptop | Dual-core processor, 4 GB RAM, 1366x768 display | Quad-core processor, 8 GB RAM, Full HD display |
| Public user desktop or laptop | Dual-core processor, 4 GB RAM | Quad-core processor, 8 GB RAM |
| Mobile user device | Android or iOS smartphone with modern browser and 2 GB RAM | 4 GB RAM or better with stable mobile data or Wi-Fi |

## 3.5.2 Software Requirements

| Category | Requirement |
|---|---|
| Operating system | Windows 10 or later, macOS, or Linux for development; any modern desktop or mobile OS for end users |
| Browser | Latest or recent versions of Chrome, Edge, Firefox, Safari, or equivalent Chromium-based browser |
| Runtime for frontend development | Node.js version 18 or later |
| Package manager | npm |
| Programming languages | TypeScript for frontend, SQL for database logic, Python for worker automation |
| Frameworks and libraries | React, Vite, Tailwind CSS, Supabase client libraries, React Router, TanStack Query |
| Database platform | PostgreSQL through Supabase |
| Development tools | Visual Studio Code, Git, ESLint, Vitest |

## 3.5.3 Network and Deployment Requirements

| Requirement | Description |
|---|---|
| Internet connectivity | Required for authentication, database access, document retrieval, and cloud-hosted operation |
| HTTPS-enabled deployment | Recommended for secure public and admin access |
| Cloud backend | Requires a configured Supabase project for authentication, database, storage, and policies |
| Web hosting | Requires a deployment environment such as Vercel for frontend hosting |
| Optional external integration | Google Forms and Google Sheets may be required when registration sync is enabled |
| Map and location services | Internet access is required for map tiles and geocoding helpers |

## 3.5.4 User and Access Requirements

| User Type | Access Requirement |
|---|---|
| Public viewer | No login required for public information modules |
| Youth user | Authenticated account required for registration, profile, and participation history features |
| Admin or staff user | Authorized role assignment required for admin modules and protected data operations |
| System maintainer | Environment variables, database migration access, and deployment credentials required |

## Summary

These system requirements are more suitable for LYDO Connect than the template's marketplace-specific setup because they reflect a responsive cloud-based governance platform with optional external registration synchronization.
