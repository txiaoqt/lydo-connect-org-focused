# 3.5 System Requirements

This section presents the hardware, software, network, and access requirements for the current LYDO Connect application workflow.

## 3.5.1 Hardware Requirements

| Environment | Minimum Requirement | Recommended Requirement |
|---|---|---|
| Developer workstation | Dual-core processor, 8 GB RAM, 10 GB free storage | Quad-core processor, 16 GB RAM, SSD storage |
| User device | Modern smartphone, tablet, laptop, or desktop with a current browser | Larger screen with stable internet connection |

## 3.5.2 Software Requirements

| Category | Requirement |
|---|---|
| Operating system | Windows, macOS, Linux, Android, or iOS with a modern browser |
| Browser | Recent Chrome, Edge, Firefox, Safari, or equivalent Chromium-based browser |
| Runtime | Node.js 18 or later |
| Package manager | npm |
| Languages | TypeScript, SQL |
| Frameworks and libraries | React, Vite, Tailwind CSS, React Router, Supabase client libraries, shadcn/ui |
| Database platform | PostgreSQL through Supabase |
| Development tools | Visual Studio Code, Git, ESLint, Vitest |

## 3.5.3 Network and Deployment Requirements

| Requirement | Description |
|---|---|
| Internet connectivity | Required for authentication, file loading, preview pages, and database access |
| HTTPS deployment | Recommended for secure access |
| Supabase project | Required for authentication, database, and storage services |
| Frontend hosting | Required for deployment of the React application |

## 3.5.4 User and Access Requirements

| User Type | Access Requirement |
|---|---|
| Public visitor | No login required for public news and transparency pages |
| Organization user | Authenticated account and active policy acceptance required for protected workflows |
| Admin user | Authorized admin account required for admin routes and content management |

## 3.5.5 Data and Source Requirements

| Requirement | Description |
|---|---|
| Organization profile data | Must be stored as user-linked profile records |
| Document files | Must be supported through file upload and preview |
| Budget and liquidation records | Must be stored with status fields and linked attachments |
| News and transparency records | Must support publication metadata and preview links |
| Template records | Must store active document template metadata and file URLs |

## Summary

These system requirements reflect the actual application workflow and exclude modules that are not present in the current build.
