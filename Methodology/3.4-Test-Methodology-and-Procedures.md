# 3.4 Test Methodology and Procedures

Testing is an essential part of the development of LYDO Connect because the system handles organization registration, compliance document submission, budget request processing, liquidation reporting, YPOP validation, YORP-related registry monitoring, policy agreement enforcement, notifications, templates, and administrative governance records. The purpose of the test methodology is to verify that each module functions correctly, integrates properly with other parts of the system, protects sensitive data, and provides a usable experience for organization users, LYDO administrators, management representatives, and evaluators.

## 3.4.1 Unit Testing

Unit testing will be conducted to verify the correctness of individual functions, components, and modules of LYDO Connect. These tests will focus on smaller units such as login validation, policy agreement handling, organization profile forms, compliance document upload helpers, OCR-related validation utilities, budget request form validation, liquidation status checks, YPOP scoring logic, YPOP submission helpers, template management functions, notification helpers, and activity log utilities.

Each unit will be tested in isolation to confirm that it accepts valid input, rejects invalid input, produces the expected output, and handles errors properly. For example, tests may verify that Philippine mobile numbers are validated correctly, required registration fields display proper error messages, policy acceptance states are handled consistently, budget amounts are computed safely, liquidation access remains locked until the required budget status is reached, and YPOP scoring rules return correct totals from project counts and city-led attendance records.

To support efficient unit testing, the developers may use available testing frameworks and libraries such as Vitest and React Testing Library. These tools allow the definition and execution of repeatable test cases, making it easier to identify errors early and maintain the quality of the codebase as the system grows.

## 3.4.2 Integration Testing

Integration testing will evaluate how the different modules of LYDO Connect work together. Since the system connects authenticated accounts, organization profiles, Supabase database records, storage, policy agreement records, compliance submissions, budget workflows, liquidation workflows, YPOP validation records, YORP-related organization fields, template management, notifications, and activity logs, it is important to verify that data are exchanged correctly between related components.

The integration tests will include checking the connection between user accounts and organization profiles, policy agreement records and protected routes, required document templates and document submission files, budget request records and attached files, approved or released budgets and liquidation record availability, YPOP periods and city activities, YPOP entries and uploaded proof files, YORP-related organization registry fields and admin updates, and administrative actions and their corresponding notifications or audit logs.

These tests will verify data flow, error handling, boundary cases, and exception scenarios. Examples include missing uploaded files, invalid record references, rejected or revised submissions, budget approval actions that should no longer be available after the record moves to another status, failed database updates, and records that are saved successfully in one module but must also appear correctly in monitoring, notification, or review screens.

## 3.4.3 System Testing

System testing will evaluate LYDO Connect as a complete and integrated application. This phase will test whether the full system satisfies the documented functional and non-functional requirements. The system will be tested through realistic scenarios that represent actual usage by organization users, LYDO administrators, and management or evaluator roles.

The test cases will cover end-to-end workflows such as signing in, accepting the active Terms of Service and Privacy Policy, completing an organization profile, submitting compliance documents, previewing uploaded files, creating and submitting a budget request, receiving a revision or approval decision, uploading liquidation files after the required budget status is reached, creating or updating a YPOP entry, uploading YPOP proof files, reviewing YPOP validation results, checking notifications, and browsing news releases and portal updates.

For the admin interface, system testing will cover dashboard viewing, registration and document review, organization profile verification, budget approval or revision handling, hard-copy recording, cash release, liquidation monitoring, YPOP period and activity management, YPOP validation decisions, YORP-related organization registry updates, template maintenance, news release maintenance, notification review, activity log inspection, and budget monitoring summaries by district and barangay.

The goal of system testing is to confirm that the application behaves consistently under normal usage, maintains data integrity, and supports the expected workflows before final evaluation or deployment.

## 3.4.4 Acceptance Testing

Acceptance testing will determine whether LYDO Connect meets the needs and expectations of its intended users and stakeholders. This testing phase provides organization users, LYDO administrators, management representatives, and evaluators an opportunity to assess whether the system is understandable, useful, and aligned with the objectives of the study.

During this phase, selected participants will perform representative tasks based on their roles. Organization users may be asked to sign in, accept the policy agreement, complete their organization profile, submit required compliance files, create a budget request, review request status, upload liquidation files when allowed, submit a YPOP entry, and read notifications or portal updates. LYDO administrators may be asked to review organization records, approve or request revision of compliance files, process budget requests, manage liquidation records, maintain YPOP and YORP-related records, manage templates and news content, and inspect activity logs.

Feedback collection methods such as structured questionnaires, interviews, and open-ended comments may be used to gather insights from participants. The collected feedback will be analyzed to identify usability concerns, missing requirements, workflow issues, and possible enhancements. The system may be considered acceptable if the major workflows are completed successfully and the evaluation results fall within the acceptable or highly acceptable range.

## 3.4.5 Performance Testing

Performance testing will assess whether LYDO Connect can provide a smooth and responsive experience during normal and expected usage. Since the system includes authenticated access, compliance file upload and preview, budget and liquidation tracking, YPOP record handling, dashboard summaries, table filtering, notification loading, and admin review pages, it must remain responsive while handling common organization-user and administrator tasks.

Load testing may be performed to determine how the system behaves when multiple users access protected pages, submit records, or view database-backed information. Stress testing may also be conducted to observe how the system responds when the amount of data or number of simultaneous actions increases beyond normal expectations.

Performance checks may include page loading time, upload and preview responsiveness, organization profile save time, budget request submission speed, admin table loading time, dashboard data retrieval, barangay and district filtering responsiveness, notification retrieval behavior, and the ability of the system to handle common user and administrator workflows without noticeable delay. Any identified performance bottlenecks will be reviewed and optimized to improve responsiveness, stability, and scalability.

## 3.4.6 Security Testing

Security testing will be conducted to protect organization information, compliance records, financial workflow records, YPOP submissions, policy acceptance records, notifications, and administrative governance data. Since LYDO Connect includes authenticated accounts and role-based administrative features, security testing is necessary to ensure that only authorized users can access protected workflows.

The security tests will include checking authentication behavior, role-based redirection, admin route protection, sign-out behavior, policy agreement enforcement for authenticated non-admin users, and restrictions on admin-only actions such as review, approval, release, template maintenance, YORP-related updates, and activity-log inspection. Input validation will also be tested using invalid emails, invalid contact numbers, missing required fields, invalid file submissions, invalid dates, invalid amounts, malformed links, and improper status transitions.

Security testing will also examine database constraints, audit logging, and data integrity rules. The system should prevent duplicate or impossible records, protect administrative actions, and record important create, update, and review operations in the activity logs. Vulnerability scanning, code review, and penetration-style checks may be performed when available to identify weaknesses before deployment.

## 3.4.7 Usability Testing

Usability testing will evaluate how easily intended users can understand and use LYDO Connect. The testing will observe users as they interact with the organization user portal and admin portal, focusing on whether navigation, labels, buttons, forms, tables, status badges, upload controls, and feedback messages are clear and consistent.

For the organization user portal, usability tasks may include signing in, accepting the policy agreement, updating the organization profile, uploading compliance files, creating a budget request, monitoring request status, submitting liquidation files, submitting a YPOP entry, and reading notifications or portal updates. For the admin portal, usability tasks may include reviewing registrations, checking compliance submissions, processing budgets, monitoring liquidation, maintaining YPOP records, updating YORP-related organization fields, filtering budget summaries, managing templates, and reviewing activity logs.

The main objective of usability testing is to identify user experience issues, confusing workflows, unclear labels, difficult forms, or layout problems. User feedback will help improve the interface so that the system becomes more intuitive, efficient, readable, and practical for the intended users.

## 3.4.8 Compatibility Testing

Compatibility testing will verify whether LYDO Connect works properly across the platforms and devices commonly used by the target users. The system will be tested on current desktop and mobile browsers to confirm that pages, forms, upload controls, navigation menus, tables, previews, dialogs, dashboards, and review screens remain usable and readable across different screen sizes.

Testing will include checking browser compatibility, responsive layout behavior, mobile navigation, form controls, file preview behavior, admin table usability, filter interactions, status displays, and modal dialogs for review actions. The goal is to ensure that organization users, LYDO administrators, management representatives, and evaluators can access the system consistently whether they are using a desktop computer, laptop, tablet, or mobile phone.

Any compatibility issues discovered during testing will be documented and addressed to improve accessibility, reliability, and consistency of the user experience across supported environments.
