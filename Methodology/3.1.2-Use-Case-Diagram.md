# 3.1.2 Use Case Diagram

This section presents one general UML use case diagram for LYDO Connect. The diagram consolidates the public portal, authenticated youth/citizen workflows, and authorized admin portal workflows into a single system boundary. It is based on the current application modules: programs, events, organizations, profile and participation history, citizen desk, transparency registry, transparency board, financial disclosure, barangay map data, dashboard, registration monitoring, user and role administration, and audit logs.

![Figure 1. General Use Case Diagram of LYDO Connect](./diagrams/use-case-overall-lydo-connect.svg)

## Diagram Notes

- The diagram models portal-local registration and record management only.
- External attendance automation is intentionally outside the scope of this use case diagram.
- Solid actor lines show direct participation in a use case.
- Dashed `<<include>>` arrows show required reusable behavior.
- Dashed `<<extend>>` arrows show optional or conditional behavior.
- Hollow-triangle arrows show actor or use-case parent-child generalization.
