# 3.1.2 Use Case Diagram

This section presents one general UML use case diagram for LYDO Connect. The diagram consolidates the sixteen main use cases documented in the use case report into a single system boundary. It covers account access, active policy agreement, public portal viewing, youth participation services, Pasig youth organization information viewing, organization information posting, citizen desk services, admin dashboard access, youth record management, local registration monitoring with external Google Forms/Sheets sync support, transparency and financial management, citizen ticket management, user and role administration, and audit log review.

![Figure 1. General Use Case Diagram of LYDO Connect](./diagrams/use-case-overall-lydo-connect.svg)

## Diagram Notes

- Orange use case ovals represent the sixteen main use cases from UC-01 to UC-16 in the use case report.
- White use case ovals represent supporting included or extended behaviors listed inside the use case reports, such as validation, policy agreement checking, source inspection, CSV export, sync retry, and external sheet preview.
- The diagram models portal-local registration as the source of record, with external registration sync treated as an administrative monitoring and integration behavior.
- Organization behavior is read-centric for public users: users inspect organization information and verified references, without organization join or enrollment actions.
- Solid actor lines show direct participation in a use case.
- Black dashed `<<include>>` arrows show required supporting behavior.
- Blue dashed `<<extend>>` arrows show optional or conditional supporting behavior.
- Hollow-triangle arrows show actor generalization, where Youth User and Citizen inherit public browsing access from Public Viewer.
