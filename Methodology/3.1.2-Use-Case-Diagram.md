# 3.1.2 Use Case Diagram

This section presents the use case diagram for the current LYDO Connect workflow. The diagram is centered on three actors: **Public Visitor**, **Organization User**, and **LYDO Admin**.

## Figure 1. General Use Case Diagram of LYDO Connect

![Figure 1. General Use Case Diagram of LYDO Connect](./diagrams/use-case-overall-lydo-connect.svg)

## Diagram Notes

- The **Public Visitor** can sign up, sign in, view published news releases, and view public transparency posts.
- The **Organization User** can sign in, sign out, accept the active policy, manage the organization profile, submit required documents, create and update budget requests, upload liquidation files after approval, view notifications, and open news preview pages.
- The **LYDO Admin** can review organization records, validate documents, review and update budget requests, monitor liquidation records, manage news releases, manage transparency posts, manage required document types, and review notifications and activity logs.
- Include relationships represent required supporting actions such as authentication, policy acceptance, validation, file upload, and status updates.
- Extend relationships represent optional preview actions, file replacement, publication, or status changes.

## Scope Note

Legacy modules from earlier drafts are intentionally excluded from the current use-case scope.
