# 3.1.2 Use Case Diagram

This section presents the use case diagram for the current LYDO Connect workflow. The diagram focuses on the two primary actors who perform the core operations of the system: **Organization User** and **LYDO Admin**.

## Figure 1. General Use Case Diagram of LYDO Connect

![Figure 1. General Use Case Diagram of LYDO Connect](./diagrams/use-case-overall-lydo-connect.svg)

## Diagram Notes

- The **Organization User** performs the core organization-side operations shown in the general diagram: account login, organization profile management, registration and compliance submission, budget and liquidation submission, YPOP entry management, request-status monitoring, published update viewing, and notification receipt.
- The **LYDO Admin** performs the core administrative operations shown in the general diagram: admin login, registration review, compliance review, budget review, liquidation review, budget allocation and utilization monitoring, YPOP and YORR validation, content posting, notification sending, and audit-log monitoring.
- The blue use cases represent the main actor-facing system goals included in the simplified general diagram.
- The white use cases represent only the supporting included and extending actions that remain necessary in the general diagram.
- `<<include>>` relationships represent required supporting actions such as authentication, document upload, request-status updating, and organization-user notification.
- `<<extend>>` relationships represent only the selected optional actions kept in the diagram: request revision, document resubmission, budget discrepancy flagging, and attached-remarks viewing.

## Detailed Use Cases

- Login appears separately for the organization user and the admin, but both include the same authentication support use case.
- Organization-side submission workflows share a common included document-upload action in the simplified general diagram.
- Administrative review workflows share the request-status update action whenever the admin completes a registration, compliance, budget, liquidation, or YPOP/YORR review decision.
- Notification sending is modeled as an admin-facing main use case that includes notifying the organization user.
- Only a few optional extensions are shown in the general diagram so that the figure remains readable while still capturing important conditional behaviors.

## Scope Note

The general use case diagram is limited to the simplified core authenticated operations of LYDO Connect. It does not attempt to show every route, every validation detail, or every possible review outcome. More specific revision, resubmission, discrepancy, and remarks behavior is handled in the use case report text rather than being expanded into a crowded main diagram.
