# 3.1.2 Use Case Diagram

This section presents the use case diagram for the current LYDO Connect workflow. The diagram is centered on two actors: **Organization User** and **LYDO Admin**.

## Figure 1. General Use Case Diagram of LYDO Connect

![Figure 1. General Use Case Diagram of LYDO Connect](./diagrams/use-case-overall-lydo-connect.svg)

## Diagram Notes

- The **Organization User** can sign in, sign out, accept the active policy, manage the organization profile, submit required documents, create and update budget requests, upload budget supporting files, view notifications, and submit liquidation files after approval.
- The **LYDO Admin** can review organization records, validate documents, review and update budget requests, monitor liquidation records, monitor barangay allocation summaries, manage news releases, manage required document types, and review notifications and activity logs.
- Include relationships represent required supporting actions such as user authentication, policy agreement loading, record validation, and file validation.
- Extend relationships represent conditional or optional outcomes such as login errors, file submission errors, liquidation access after budget approval, and budget allocation filtering by district or barangay.

## Detailed Use Cases

- Authentication and policy agreement act as the entry point for protected organization workflows.
- Document submission depends on the organization profile and the active required document list.
- Budget request processing includes draft creation, submission, approval, release, and liquidation unlocking.
- Barangay budget monitoring is an admin-only reporting view that groups released amounts by district and barangay.
- News release preview is handled inside the authenticated workflow, while transparency posting is no longer shown as a primary admin use case in this diagram.

## Scope Note

Legacy modules from earlier drafts are intentionally excluded from the current use-case scope. The current diagram emphasizes the live site workflow, especially the budget monitoring and barangay allocation reporting that replaced the older transparency-post focus on the admin side. There is no public-visitor actor in the current scope.
