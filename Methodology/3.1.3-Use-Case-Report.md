# 3.1.3 Use Case Report

This section summarizes the core operations supported by the current LYDO Connect workflow. The report reflects the validated site scope for the Organization User and LYDO Admin / Staff portals, then includes a separate planned section for LYDO Super Admin use cases that are intended for later implementation.

## Organization User Use Cases

## Table 1. Use Case Report of Log in to Account

| Field | Details |
|---|---|
| USE CASE ID | UC-01 |
| USE CASE NAME | Log in to Account |
| SCENARIO | An organization user signs in to access the protected organization portal. |
| TRIGGERING EVENT | The organization user opens the sign-in page and submits account credentials. |
| BRIEF DESCRIPTION | This use case authenticates the organization user and grants access to the organization-facing workflows of LYDO Connect. |
| ACTORS | Organization User |
| INCLUDE USE CASE | Authenticate User |
| EXTEND USE CASE | None |
| PRE-CONDITIONS | The organization user already has a valid account. |
| POST-CONDITIONS | The organization user is authenticated and may continue to the protected portal. |

### Flow of Events

| ACTOR | SYSTEM |
|---|---|
| 1. The organization user opens the sign-in page. | 1. The system displays the login form. |
| 2. The organization user enters the account credentials. | 2. The system validates the submitted credentials. |
| 3. The organization user submits the form. | 3. The system authenticates the account. |
| 4. The organization user waits for the result. | 4. The system opens the protected organization workflow after successful authentication. |

| ALTERNATIVE FLOW | If the credentials are invalid, the system displays an error and keeps the user on the sign-in page. |
|---|---|
| EXCEPTION CONDITIONS | Invalid credentials, missing fields, or authentication service failure may prevent access. |
