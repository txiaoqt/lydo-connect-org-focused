# 3.1.3 Use Case Report

This section presents the use case reports of the major functions of LYDO Connect using the format required by the manuscript template. Each use case report identifies the scenario, triggering event, actors, included use cases, pre-conditions, post-conditions, flow of events, alternative flow, and exception conditions.

## Table X. Use Case Report of Sign Up

| USE CASE ID: | UC-01 |
|---|---|
| USE CASE NAME: | Sign Up |
| SCENARIO: | Normal |
| TRIGGERING EVENT: | The user selects the `Sign Up` option from the portal. |
| BRIEF DESCRIPTION: | The system allows a new youth user or citizen to create an account by entering required personal and authentication details. |
| ACTORS: | Youth User, Citizen |
| INCLUDE USE CASE: | Encode User Details; Validate Registration Data |
| EXTEND USE CASE: | None |
| PRE-CONDITIONS: | The user has internet access and a valid email address. |
| POST-CONDITIONS: | A new account is created and the user may proceed to sign in. |

### FLOW OF EVENTS

| ACTOR | SYSTEM |
|---|---|
| 1. Opens the sign up page. | 1. Displays the sign up form. |
| 2. Enters full name, email, password, contact number, and barangay. | 2. Accepts the user input and prepares it for validation. |
| 3. Submits the completed registration form. | 3. Validates the required fields and account data. |
| 4. Waits for registration confirmation. | 4. Creates the account, stores the profile metadata, and assigns the default youth role. |
| 5. Proceeds to the sign in page after successful registration. | 5. Displays success feedback and next-step instructions. |

| ALTERNATIVE FLOW: | If email confirmation is required, the system informs the user to verify the email address before full account use. |
|---|---|
| EXCEPTION CONDITIONS: | The entered email may already exist, the password may fail validation rules, or the network request may fail. |
