# 3.1.3 Use Case Report

This section presents the updated use case reports for the LYDO Connect System based on the revised use case diagram. The diagram identifies four actors: **Public User**, **Youth User**, **LYDO Staff**, and **LYDO Super Admin**. The system scope focuses on public portal access, youth account registration and login, youth program/event registration, participation tracking, youth service requests, admin content management, participation monitoring, reporting, user account and role management, and audit log review.

## Table X. Use Case Report of Register

| USE CASE ID: | UC-01 |
|---|---|
| USE CASE NAME: | Register |
| SCENARIO: | A public user creates an account to access youth user services. |
| TRIGGERING EVENT: | The Public User selects the registration option. |
| BRIEF DESCRIPTION: | This use case allows a public user to submit account details, validate the provided information, and complete email verification before becoming a registered youth user. |
| ACTORS: | Public User |
| INCLUDE USE CASE: | Validate Credentials; Email Verification |
| EXTEND USE CASE: | Registration Error |
| PRE-CONDITIONS: | The Public User has access to the registration page and has the required account information. |
| POST-CONDITIONS: | A user account is created and the user may proceed to log in. |

### FLOW OF EVENTS

| ACTOR | SYSTEM |
|---|---|
| 1. The Public User opens the registration page. | 1. The system displays the registration form. |
| 2. The Public User enters the required account details. | 2. The system validates the credentials and required fields. |
| 3. The Public User submits the form. | 3. The system sends or processes email verification. |
| 4. The Public User completes verification. | 4. The system creates the account and displays a registration confirmation. |

| ALTERNATIVE FLOW: | If validation or email verification fails, the system displays a registration error and allows the user to correct the information. |
|---|---|
| EXCEPTION CONDITIONS: | Duplicate email, invalid contact details, missing required fields, weak password, failed email verification, or authentication service error may prevent registration. |
