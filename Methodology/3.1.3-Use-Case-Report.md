# 3.1.3 Use Case Report

This section summarizes the core operations shown in the general use case diagram of LYDO Connect. The report is limited to the authenticated organization workflow and the LYDO administrative workflow that are currently central to the implemented system.

## Table 1. Use Case Report of Login

| Field | Details |
|---|---|
| USE CASE ID | UC-01 |
| USE CASE NAME | Login |
| SCENARIO | An organization user or LYDO Admin signs in to access the proper portal. |
| TRIGGERING EVENT | The actor opens the sign-in page and submits account credentials. |
| BRIEF DESCRIPTION | This use case authenticates the actor, checks the role, loads the active policy for non-admin users, and opens the user or admin portal. |
| ACTORS | Organization User, LYDO Admin |
| INCLUDE USE CASE | Authenticate User |
| EXTEND USE CASE | Load Active Policy |
| PRE-CONDITIONS | The actor already has a valid account. |
| POST-CONDITIONS | The actor is routed to the correct portal or returned to the sign-in form with feedback. |

### Flow of Events

| ACTOR | SYSTEM |
|---|---|
| 1. The actor opens the sign-in page. | 1. The system displays the login form. |
| 2. The actor enters the required credentials. | 2. The system validates and authenticates the account. |
| 3. The actor submits the form. | 3. The system checks whether the actor is an organization user or an admin. |
| 4. The actor continues to the next step. | 4. The system opens the admin portal directly or loads the active policy before entering the user portal. |

| ALTERNATIVE FLOW | If authentication fails, the system displays an error and keeps the user on the sign-in page. |
|---|---|
| EXCEPTION CONDITIONS | Invalid credentials, missing fields, or authentication service failure may prevent access. |

## Summary

The current use case report focuses only on the authenticated core operations represented in the general use case diagram. These operations cover login, organization-side compliance and YPOP workflows, and the main administrative workflows for review, monitoring, content maintenance, and audit-oriented tracking.
