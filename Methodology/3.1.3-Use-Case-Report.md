# 3.1.3 Use Case Report

This section summarizes the current implemented use cases of LYDO Connect. The report follows the live workflow in the application and includes the public surface, the protected organization workflow, and the expanded admin workflow.

## Table 1. Use Case Report of Browse Public Information

| Field | Details |
|---|---|
| USE CASE ID | UC-01 |
| USE CASE NAME | Browse Public Information |
| SCENARIO | A public visitor opens the public website to read general information about LYDO Connect and related pages. |
| TRIGGERING EVENT | The visitor opens the landing page or another public route. |
| BRIEF DESCRIPTION | This use case covers the public pages available without authentication, such as the home page, about page, frequently asked questions, contacts, legal policy pages, and site map. |
| ACTORS | Public Visitor |
| INCLUDE USE CASE | Load Public Content |
| EXTEND USE CASE | Open News Release Record |
| PRE-CONDITIONS | The public site is reachable. |
| POST-CONDITIONS | The visitor reads the requested public information. |

### Flow of Events

| ACTOR | SYSTEM |
|---|---|
| 1. The visitor opens the public site. | 1. The system displays the landing page or requested public page. |
| 2. The visitor navigates to pages such as About, FAQs, Contacts, Terms, Privacy, or Site Map. | 2. The system loads the selected content. |
| 3. The visitor reviews the information. | 3. The system keeps the selected page available for further navigation. |
| 4. The visitor may proceed to account access or leave the site. | 4. The system offers links to sign in, sign up, and published records. |

| ALTERNATIVE FLOW | If a requested route does not exist, the system shows the not-found page or redirects based on the active surface. |
|---|---|
| EXCEPTION CONDITIONS | Missing page content or routing errors may prevent a public page from loading correctly. |

## Summary

The current use-case scope now covers the real application workflow across all three actors. It includes public browsing and account access, organization-side compliance and YPOP processing, and the full admin workflow for registrations, finance, YPOP, YORP registry, content, templates, notifications, and activity monitoring. Legacy transparency-post administration and redirect-only routes are excluded because they no longer represent distinct current workflows.
