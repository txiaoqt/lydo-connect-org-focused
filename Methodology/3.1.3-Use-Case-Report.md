# 3.1.3 Use Case Report

This section presents the use case reports for the main LYDO Connect functionalities shown in the updated general use case diagram. The report follows the reference format by identifying the use case ID, use case name, scenario, triggering event, brief description, actors, included use cases, extended use cases, pre-conditions, post-conditions, flow of events, alternative flow, and exception conditions.

The updated use case reports are aligned with the current general diagram scope: account access, active policy agreement, public portal viewing, youth participation services, organization information posting, citizen desk services, admin dashboard access, administrative record management, registration monitoring with Google Forms/Sheets sync support, transparency and financial management, user and role administration, and audit log review. Supporting behaviors such as validation, active session checking, policy acceptance checking, searching, filtering, exporting, sync retry, external sheet preview, and status updating are treated as included or extended use cases rather than separate main use case reports.

## Table X. Use Case Report of Register Account

| USE CASE ID: | UC-01 |
|---|---|
| USE CASE NAME: | Register Account |
| SCENARIO: | To gain access to authenticated youth and citizen services, a new user submits the required account registration details. |
| TRIGGERING EVENT: | Initiated when the actor selects the registration option from the portal. |
| BRIEF DESCRIPTION: | This use case allows a youth user or citizen to create a LYDO Connect account by providing account and profile details such as email, full name, contact number, and barangay. Once the details are validated, the system creates the account and prepares the user for log in and policy agreement checking. |
| ACTORS: | Youth User, Citizen |
| INCLUDE USE CASE: | Validate Account Details |
| EXTEND USE CASE: | N/A |
| PRE-CONDITIONS: | The actor has access to the portal and has the required registration information. |
| POST-CONDITIONS: | A new account and profile are created, and the actor may proceed to log in and accept the active Terms of Service and Privacy Policy when required. |

### FLOW OF EVENTS

| ACTOR | SYSTEM |
|---|---|
| 1. The actor opens the registration page. | 1. The system displays the account registration form. |
| 2. The actor enters the required account and profile details and submits the form. | 2. The system validates the submitted account details. |
| 3. The actor reviews the registration result. | 3. The system creates the account, stores profile metadata, and displays a confirmation message. |

| ALTERNATIVE FLOW: | If required details are incomplete or invalid, the actor corrects the highlighted fields and submits the form again. |
|---|---|
| EXCEPTION CONDITIONS: | Duplicate email address, invalid account details, weak password, invalid contact number, missing barangay data, or unavailable authentication service may prevent registration. |
