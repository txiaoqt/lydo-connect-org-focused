# 3.3 Development Methodology

The development methodology for LYDO Connect follows a **hybrid Design Science Research (DSR) and Rapid Application Development (RAD)** approach. This combination is appropriate because the study aims both to produce a working artifact that solves a real governance problem and to refine that artifact through iterative design and stakeholder feedback.

## Why DSR Is Appropriate

DSR is suitable because the study is centered on the creation of an information system artifact. LYDO Connect is not only being described; it is being designed, developed, demonstrated, and evaluated as a solution to fragmented youth governance, transparency, and citizen service processes.

Through DSR, the study can:

- identify the real operational problem in LYDO-related information and transparency management;
- define solution objectives based on stakeholder and governance needs;
- build the system artifact;
- demonstrate how the artifact addresses the identified problem; and
- evaluate the artifact using quality and usability criteria.

## Why RAD Is Appropriate

RAD is suitable because the system contains multiple modules that benefit from quick prototyping and repeated improvement. The current repository itself shows iterative refinement of database migrations, registration flows, transparency modules, and admin tooling. This makes RAD more realistic than a rigid linear model.

Through RAD, the study can:

- create working prototypes early;
- refine modules based on feedback;
- improve forms, dashboards, and record management quickly;
- support faster adjustment of database structures and validation rules; and
- shorten development cycles for both public and admin interfaces.

## Hybrid Use in This Study

In this study, DSR provides the **research framework**, while RAD provides the **software construction strategy**. DSR explains why the artifact is being developed and how it will be evaluated. RAD explains how the system is rapidly planned, prototyped, revised, implemented, and refined.

## Appropriateness Note

The template's original waterfall discussion is not the best fit for LYDO Connect. A hybrid DSR and RAD methodology is more aligned with the actual development style, the iterative schema changes, and the evolving public and administrative workflows present in the system.
