# 3.0 Methodology

This section presents the methodology for developing **LYDO Connect**, a web-based organization compliance and content management system for local youth development operations. The current application workflow centers on authenticated user access, policy agreement, organization profile management, document submission, budget request processing, liquidation reporting, news release previewing, public transparency posting, notifications, template management, and admin review.

## Research Design

The study uses a **descriptive-developmental research design**. It is descriptive because it identifies the needs, records, and operational tasks required by the users and administrators of the system. It is developmental because it focuses on designing and refining a working web application that supports the actual LYDO Connect workflow in the repository.

The study is guided by **Design Science Research (DSR)** and **Rapid Application Development (RAD)**. DSR provides the research structure for defining the problem, specifying the solution, building the artifact, and evaluating its usefulness. RAD supports iterative implementation, rapid feedback, and continuous improvement of the interface and workflow behavior.

## SDLC Model

The study follows a **modified System Development Life Cycle (SDLC)** using the RAD approach. This fits the current application because the system is composed of multiple interconnected modules that require frequent refinement. The development cycle covers requirements analysis, interface design, database and API integration, prototype implementation, testing, revision, and evaluation.

## Data-Gathering Methods

Primary data are gathered through consultation, interview, and evaluation feedback from intended users, administrators, and qualified evaluators. These inputs define the functional needs of the system and validate whether the implemented workflow matches the operational requirements.

Secondary data are gathered through document review of policy references, local office records, transparency requirements, and system-related materials that support the actual app workflow. These sources inform the records, fields, and content types supported by the application.

## System Development Approach

The study combines **DSR as the research framework** and **RAD as the software development strategy**. The artifact is built incrementally, verified through testing, and revised based on feedback.

For LYDO Connect, the current workflow supports:
- authenticated user access and policy acceptance
- organization profile management
- document submission with uploaded files and preview
- budget request CRUD and file attachments
- liquidation access after budget approval
- news release cards with Facebook preview pages
- public transparency posts
- template/document type management
- admin monitoring, review, and audit-friendly content updates

## Summary

Overall, the methodology is anchored on a descriptive-developmental design, a modified SDLC using RAD, and iterative validation of the current app workflow. This approach is appropriate because it supports the development and review of the actual modules already implemented in LYDO Connect.
