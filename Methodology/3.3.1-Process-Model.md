# 3.3.1 Process Model

LYDO Connect follows an iterative process model:

1. Define the current workflow scope.
2. Design the module or change.
3. Implement the feature in the codebase.
4. Test the feature locally.
5. Revise the workflow based on issues found.
6. Deploy or commit the updated artifact.

This process is repeated for each module in the current app workflow, including authentication, organization profiles, document submission, budget requests, liquidation, news releases, transparency posts, templates, and admin review.
