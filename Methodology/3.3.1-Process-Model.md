# 3.3.1 Process Model

LYDO Connect follows an iterative process model:

1. Define the current workflow scope.
2. Design the module or change against the live workflow.
3. Implement the feature in the codebase and Supabase schema when needed.
4. Test the feature locally.
5. Revise the workflow based on issues found.
6. Commit or deploy the updated artifact.

This process is repeated for each module in the current app workflow, including authentication, policy acceptance, organization profiles, document submission, budget requests, liquidation, barangay budget allocation monitoring, news releases, transparency posts, required document types, notifications, and admin review.
