# 3.7 Evaluation Plan

The evaluation will measure whether LYDO Connect is usable, reliable, secure, and aligned with the current workflow.

## 3.7.1 Evaluation Activities

- user acceptance testing for organization users and admins
- functional workflow testing for the current modules
- record review for documents, budget requests, liquidations, news releases, transparency posts, templates, notifications, and logs
- record review for budget allocation summaries by barangay and district
- usability survey for respondents who use the app
- performance observation for page loading and preview behavior
- security and access control checks for protected routes and admin actions

## 3.7.2 Respondents and Participants

The evaluation may involve organization users, LYDO admins, and qualified reviewers or evaluators.

## 3.7.3 Evaluation Instrument

The main instrument will be a Likert-scale questionnaire covering functional suitability, usability, reliability, responsiveness, security, reporting clarity, and overall acceptability.

## 3.7.4 Data Gathering Procedure

1. Prepare the evaluation task guide and questionnaire.
2. Demonstrate the current app workflow.
3. Allow participants to complete representative tasks.
4. Collect survey responses and comments.
5. Summarize the results and identify improvement points.

## 3.7.5 Data Analysis Plan

The responses may be summarized using frequency, weighted mean, and standard deviation. Open-ended comments may be grouped into common themes, especially on clarity of monitoring, filtering, and workflow feedback.

## 3.7.6 Acceptance Criteria

The system may be considered acceptable if the current workflows complete successfully, the interface is understandable on mobile and desktop, the budget allocation monitoring view shows correct barangay and district summaries, and no critical functional or security issue remains unresolved.
