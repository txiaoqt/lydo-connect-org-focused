# 3.2.4 Database Schema

The database schema describes the tables that support the current LYDO Connect workflow. The present schema is centered on authentication, policy acceptance, organization profiles, document submission, budget requests, liquidation reporting, news releases, transparency posts, templates, notifications, activity logs, and compliance remarks.

## Overall Database Schema

```mermaid
erDiagram
    auth.users ||--|| user_profiles : user_id
    auth.users ||--o{ user_roles : user_id
    roles ||--o{ user_roles : role_id
    auth.users ||--o| admin_accounts : auth_user_id
    auth.users ||--o{ user_policy_acceptance : user_id
    policy_versions ||--o{ user_policy_acceptance : policy_version_id

    auth.users ||--o{ organization_profiles : user_id
    organization_profiles ||--o{ document_submissions : organization_id
    document_submissions ||--o{ document_submission_files : submission_id
    organization_profiles ||--o{ budget_requests : organization_id
    budget_requests ||--o{ budget_request_files : budget_request_id
    budget_requests ||--o{ liquidation_reports : budget_request_id
    liquidation_reports ||--o{ liquidation_report_files : liquidation_report_id

    auth.users ||--o{ news_releases : created_by
    auth.users ||--o{ transparency_posts : created_by
    auth.users ||--o{ compliance_remarks : created_by
    auth.users ||--o{ notifications : user_id
    auth.users ||--o{ activity_logs : actor_user_id
    auth.users ||--o{ templates : created_by
```
