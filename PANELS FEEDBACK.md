# PANELS FEEDBACK

## Context

This document summarizes the consultation with the panelists regarding the current site concept and the direction that the system must take.

Panelists:

- Woman 1
- Man 1

## Overall Direction From The Panels

The strongest message from the panel was that the site should be **organization-centric**, not only youth-information-centric.

What they want is a system where organizations can actually transact with the site and with the Local Youth Development Office workflow. The site should not feel like a simple public directory or static website.

Their main expectation is that the platform should support:

- digital submission of documents by organizations
- validation or checking of submitted requirements
- budget-related transparency
- public disclosure of reports and compliance status
- monitoring of how budget is used
- visibility of what happens when funds are not properly used

## Main Points Raised By The Panel

### 1. The system should be organization-centric

The panel repeatedly emphasized that the site should focus on organizations, especially the organizations handled by LYDO.

Their concern:

- the current prototype feels like a viewing platform only
- the system should support actual workflow and transaction
- organizations should be able to submit files, requirements, reports, and related documents through the site

### 2. The system should support budget-related workflow

The panel connected the site directly to budget transparency and utilization.

They asked:

- if an organization is given a budget, how will the system know how the budget was used?
- what documents prove that the budget was utilized properly?
- what is the process for liquidation?
- what happens if the money is not used properly?

Their point is that budget allocation alone is not enough. The system must also show:

- how the budget is released
- how it is consumed
- what supporting documents are required
- how compliance is validated

### 3. Transparency must be concrete, not vague

The panel did not want a general transparency label only.

They wanted transparency to mean something specific, such as:

- budget allocation
- utilization
- liquidation
- disclosure documents
- accomplishment reports
- compliance records
- public posting of organization status

### 4. Monitoring should be part of the system

The panel made it clear that LYDO is not only a repository of information.

The system should help LYDO monitor:

- organization submissions
- compliance status
- budget usage
- reports and accomplishments
- missing or pending documents
- public transparency records

### 5. The roles and use cases need to be clearer

The panel questioned the use cases in the prototype.

They pointed out that the system should not be framed around only individual youth users if the real workflow is about organizations, barangays, and SK units.

They asked:

- who exactly are the users?
- are they individuals, organizations, barangays, or SKs?
- what is the actual role of LYDO?

### 6. The scope must be clarified

The panel asked whether the site is meant for:

- SK per barangay
- youth organizations
- private organizations
- school-based organizations
- all organizations under LYDO

Their point was that the project scope should be clear and defensible.

### 7. The system should define required documents

The panel challenged the team to identify the exact documents needed for:

- budget request
- approval
- submission
- liquidation
- accomplishment reporting
- compliance

They wanted the process to be concrete, not just conceptual.

### 8. The system should show consequences of non-compliance

The panel asked what happens when an organization:

- does not submit required documents
- does not use the budget correctly
- fails to liquidate properly
- has missing compliance records

This means the site should not only show status, but also show remarks, deficiencies, penalties, or follow-up actions.

## Detailed Summary Of The Main Critiques

### A. The site currently feels like a website, not a transaction system

The panel said that the current prototype mostly shows information and organization visibility.

They want a real workflow:

- request
- submit
- validate
- approve
- release
- use
- liquidate
- report
- publish transparency results

### B. Budget allocation needs clearer logic

The panel wanted to know:

- where the budget comes from
- what basis is used for allocation
- how the budget is tracked
- what evidence shows it was used correctly

### C. Compliance status must have a real meaning

Statuses like:

- fully compliant
- partially compliant
- pending
- missing

should be tied to actual submitted requirements and verifiable records.

### D. Organization classification must be more precise

The panel wanted a clearer distinction among:

- SK
- barangay-based groups
- school-based organizations
- private youth organizations
- LYDO-linked organizations

### E. Individual youth users may not be the main transaction user

The panel questioned the individual user model.

Their suggestion was that the core transaction flow should likely focus on:

- organizations
- barangays
- SK units
- LYDO administrators

while individual youth users can remain for:

- information access
- complaints
- inquiries
- announcements

## What The Panel Seems To Want The Site To Become

The panel is effectively describing a system that functions like an official LYDO portal for:

- organization records
- document submission
- compliance checking
- budget transparency
- project monitoring
- accomplishment reporting
- public disclosure
- admin validation

So the site should be repositioned from:

- "a youth information and visibility site"

to:

- "an organization-centered LYDO workflow and transparency system"

## Important Clarification About The Feedback

The panel did not reject the project idea.

What they rejected was the current direction where the system is too close to:

- a static site
- a public directory
- a general youth portal

They want it to become a system with a visible process and real administrative value.

## Recommended Interpretation Of The Panel Feedback

The core interpretation is:

1. The system should manage organizations, not just display them.
2. The system should support document-based transactions.
3. The system should support budget transparency and monitoring.
4. The system should define who submits what, when, and why.
5. The system should explain how compliance is checked and what happens when it fails.
6. The system should make LYDO's monitoring role visible.

## Specific Issues Raised By Woman 1 And Man 1

### Woman 1

- asked how many organizations LYDO handles
- questioned whether the system is really for LYDO-managed organizations
- emphasized budget and transparency
- asked what happens to the budget after it is released
- asked why the site should still exist if budget tracking is removed
- questioned whether the prototype is only showing organization information

### Man 1

- asked what the exact role of Youth Development Office is
- emphasized monitoring and compliance
- asked how utilization is validated
- asked what reports and documents are required
- asked what the process is for approvals and liquidation
- questioned why the use cases are based on individuals instead of organizations

## Key Takeaway

The panel's main message is that the site should become a **real organizational transaction and monitoring platform**.

Not just:

- view information
- show profiles
- display announcements

But also:

- submit documents
- validate requirements
- monitor budget use
- publish compliance
- track utilization
- show consequences of missing or improper submissions
